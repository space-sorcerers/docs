## Создание медицинского PR

Медицинские PR включают всё, что покрывается документом Medical Workgroup. PR, затрагивающие BodySystem и связанные компоненты, поток раунда медицинского отдела и реагенты, являются медицинскими PR и должны соответствовать перечисленным здесь руководящим принципам.

Медицинские PR должны в своём описании или названии указывать, предназначено ли это изменение для текущей upstream медицинской системы или для debodying/disco-med (см. [Medical Workgroup Document](medical-workgroup)). PR без чёткого намерения может быть закрыт, если он не соответствует руководящим принципам для текущей upstream медицины или disco-med.

По любым дополнительным вопросам обращайтесь к медицинской рабочей группе через сервер Discord SS14.

### PR для текущей медицины
PR для текущей медицины обычно должны быть направлены на исправление или ребалансировку текущего поведения. Такие PR должны обосновывать своё существование; текущая медицинская система крайне несовершенна, поэтому часть описания PR должна объяснять, почему это должно быть принято вместо ожидания лучшей медицинской реализации. PR с микробалансировкой будут закрываться на усмотрение рабочей группы. Распространённые обоснования могут включать: исправления багов, эксплойты, сильно несбалансированное/мета-поведение или небольшие дополнения, которые не сильно изменяют ключевые системы.

Кроме того, такие PR не могут и не должны отменять debodying или хардкодить поведение BodySystem в другой системе. Если PR не может соответствовать этим требованиям и не может быть изменён для их соблюдения, он должен быть переработан для соответствия руководящим принципам disco-med или закрыт.

### PR для Disco-Med/Debodying
PR, предназначенный для debodying/disco-med, должен соответствовать [Medical Design Document](../medical). PR для Disco-Med и Debodying должны быть более высокого качества кода и будут подвергаться более тщательной проверке. Эти PR должны чётко объяснять в своём описании, как это продвигает медицинскую систему или способствует разрушению/реконструкции BodySystem. PR для Disco-Med и debodying не должны делать компромиссов в коде. Если PR должен пойти на компромиссы в коде, его описание должно объяснять как эти компромиссы, так и то, что необходимо для их устранения, а также помечать эти компромиссы в коде с помощью TODO.

PR, которые необходимо разбить на несколько частей, могут потребовать дизайн-документ, это особенно актуально, если выполняется любое из следующих условий:
- PR не покрывается предыдущим дизайн-документом
- PR не был предварительно одобрен участником медицинской рабочей группы
- PR отклоняется от Medical Design Document или Medical Workgroup Document
- PR имеет такой большой объём, что может потребовать разделения между несколькими контрибьюторами
- PR требует предварительного рефакторинга нескольких других систем

По дополнительным вопросам о Debodying и Disco-Med см.: [Medical Workgroup Document](medical-workgroup)

### PR для Offmed
PR для offmed не должны сливаться в master, так как offmed является тестовой веткой. Вместо этого они должны сливаться непосредственно в саму ветку offmed. Поскольку offmed является веткой тестирования функций и полностью находится под контролем медицинской рабочей группы, любой PR, предназначенный для offmed, может быть закрыт по любой причине.
