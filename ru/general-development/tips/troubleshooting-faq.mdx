## Общее

### `System.DllNotFoundException` на ARM (например, M1) маках

В настоящее время мы не поддерживаем нативный запуск SS14/Robust на ARM маках. Установите x64 .NET и запускайте клиент с ним; он должен работать через эмуляцию Rosetta 2.

## Клиент

### Ошибки YAML Linter

Запустите линтер через IDE (обычно в выпадающем меню Run & Debug). В выводе будет точный файл и строка с ошибкой. Частые причины:

- Неверные отступы (YAML чувствителен к пробелам)
- Пропущен пробел после `:`
- Список `-` там, где ожидается ключ маппинга (например, в `components:`)
- Строка с `#` или спецсимволами без кавычек — оберните в `"broken string"`

Если сомневаетесь — сравните с рабочим прототипом рядом.

### Локализация показывает ID вместо текста

Ключ FTL не нашёлся. Проверьте:
1. Ключ есть в правильном `.ftl` в `Resources/Locale/<язык>/`.
2. Ключ совпадает символ-в-символ (без лишних пробелов, регистр).
3. Вы редактируете существующий `.ftl` в нужной директории, а не создаёте дубликаты.
4. Перезапустите клиент — hot reload мог не сработать.

## Сборка

### Сборка падает после пула апстримных изменений

Попробуйте по порядку:
1. `git submodule update --init --recursive`
2. `dotnet clean && dotnet restore && dotnet build`
3. Удалите папку `.bin` и пересоберите
4. Проверьте версию .NET через `dotnet --info` — она должна совпадать с версией в `global.json`

## Сервер

### Ошибки привязки порта при попытке запустить сервер.

Некоторые коды ошибок для ctrl+f: «An attempt was made to access a socket in a way forbidden by its access permissions.», «Only one usage of each socket address (protocol/network address/port) is normally permitted.»

По умолчанию игровому серверу для запуска нужны порты 1212/UDP и 1212/TCP. Частая причина — *уже запущенный SS14 сервер, использующий эти порты*.

#### Пользователи Windows

Установка **Docker for Windows** или чего-то подобного, похоже, полностью портит диапазон динамических портов по умолчанию в вашей системе и, вдобавок, резервирует множество портов через постоянно работающий сервис. Это означает, что *даже когда Docker не запущен*, он или *любое приложение* в вашей системе может случайно решить зарегистрировать порт 1212 для общих целей.

Если вы хотите проверить, так ли это, запустите `netsh int ipv4 show dynamicportrange tcp`. По умолчанию должно быть следующее:

```
Protocol tcp Dynamic Port Range
---------------------------------
Start Port      : 49152
Number of Ports : 16384
```

Если `Start Port` нижней границы диапазона равен, скажем, 1024, ваша система испорчена AFAICT. Это можно исправить, выполнив `netsh int ipv4 set dynamicportrange tcp startport=49152 numberofports=16384` в командной строке с повышенными привилегиями, чтобы вернуть диапазон динамических портов к рекомендациям IANA, а затем перезагрузившись.

## Частые ошибки C#

### Nullable reference types

```csharp
// CS8618 — поле не допускающее null должно содержать значение
[Dependency] private readonly IEntityManager _entityManager = default!; // <-- = default! обязательно
```

Всегда добавляйте `= default!` к полям с `[Dependency]`. Движок устанавливает их до вызова `Initialize()`.

### Отсутствует using / тип не найден

Если компилятор говорит, что тип не существует, хотя он определён:

1. Проверьте, что добавлен правильный `using`.
2. Проверьте, что тип в правильном проекте (`Content.Client`, `Content.Shared` или `Content.Server`). Класс из `Content.Server` нельзя использовать из `Content.Client` или `Content.Shared`.
3. Если тип из другого проекта — добавьте project reference в `.csproj`.

### Код в неправильном проекте

Частая ошибка: вызов `IoCManager` или `Clyde` в Content.Shared. Shared-код не может ссылаться на клиентские или серверные синглтоны. Перенесите систему в `Content.Client` или `Content.Server`.

### Слепое доверие Quick Fix / подсказкам IDE

Rider и VS при Alt+Enter могут добавить using из неправильного пространства имён. Всегда проверяйте, откуда пришёл импорт.

### Код не запускается / хендлер не вызывается

Если breakpoint в обработчике события никогда не срабатывает:

1. Зарегистрирован ли компонент (`[RegisterComponent]`)?
2. Система в правильном проекте и наследует `EntitySystem`?
3. Вызвали ли `SubscribeLocalEvent` (или `SubscribeNetworkEvent`) в `Initialize()`?
4. Есть ли у сущности этот компонент? Проверьте через VV.
