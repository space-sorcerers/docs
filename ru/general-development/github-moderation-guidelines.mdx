title: "Руководство по модерированию на Github"
---

Этот документ разъясняет руководящие принципы модерирования Github в отношении Code of Conduct проекта. Модераторы Github обязаны придерживаться этих принципов как основы для действий на репозитории Github, но могут отступать от них, если считают, что это в интересах сообщества.

Примечание: если вы считаете, что эти принципы не соблюдались или от них отступили без уважительной причины, вы можете связаться со staff через [Discord](https://discord.ss14.io/) или [форум](https://forum.spacestation14.com/), обжаловать бан на Github [на форуме](https://forum.spacestation14.com/c/ban-appeals/appeals-github/38) или подать [жалобу на staff](https://forum.spacestation14.com/t/staff-complaint-instructions-and-info/31). Неправильное применение [Code of Conduct](https://github.com/space-wizards-federation/space-station-14/blob/master/CODE_OF_CONDUCT) не повлечёт дисциплинарных мер для staff, если это сделано добросовестно, и применяется только в крайних случаях, когда добросовестность отсутствует.

## Модеративные действия

### Перманентные баны

Контент, нарушающий примеры, перечисленные под пунктом «**Be careful in the words that you choose.**» в [Code of Conduct](https://github.com/space-wizards-federation/space-station-14/blob/master/CODE_OF_CONDUCT), приведёт к немедленному перманентному бану без права апелляции и сообщению в Github org.

Это также может применяться, если оскорбительный контент находится за пределами Github Space Station 14, например, на сервере Discord Wizden, форумах или других площадках, на усмотрение модератора Github. PR, созданные нарушителем, должны быть закрыты, но их содержимое может быть подхвачено другим контрибьютором и использовано в новом PR, если лицензия это позволяет.

### Другие действия

Как указано в разделе «**On Community Moderation**», контент/комментарии, создающие негостеприимную атмосферу для проекта, могут повлечь модеративные действия. Модераторы Github имеют следующие инструменты:
- Скрытие комментариев.
  - Это предпочтительно использовать для комментариев не по теме или неоскорбительных, но не вносящих вклада в обсуждение. Такие комментарии следует помечать как «Off-Topic» или «Spam»; хотя интерфейс предоставляет и другие причины для скрытия комментария, они лучше подходят для нижеследующего.
- Удаление контента и выдача бана на Github на 24 часа.
  - Для контента/комментариев, данных в неконструктивной манере (примеры перечислены в [Code of Conduct](https://github.com/space-wizards-federation/space-station-14/blob/master/CODE_OF_CONDUCT)), контент может быть удалён, даже если он частично конструктивен. Если это необходимо, нарушителю выдаётся бан на Github на 24 часа. Хотя бан может иногда казаться чрезмерным, это единственный способ напрямую сообщить пользователю, что против него были приняты меры.
- Долгосрочные баны на Github.
  - Если пользователь неоднократно нарушает Code of Conduct, может быть оправдан бан на Github на более длительный срок. Длительность должна составлять от 7 дней до 1 месяца, на усмотрение модератора Github. Всё, что дольше — это бан с правом апелляции.
- Баны с правом апелляции.
  - Бан с правом апелляции может быть выдан вместо долгосрочного бана. Такие баны могут быть [обжалованы на форумах](https://forum.spacestation14.com/c/ban-appeals/appeals-github/38) не ранее чем через 6 месяцев после выдачи.

Из-за ограничений инструментов модерации Github, пострадавший пользователь может получить только уведомление о том, что против него было предпринято действие, но не о сроке или причине. Это возможно только через опцию «Block» в оскорбительном комментарии на Github, если это доступно. Если пострадавший пользователь запрашивает информацию о своём бане через один из официальных каналов, ему должна быть предоставлена причина.
